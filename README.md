# نظام التسوية البنكية الآلية | Automated Bank Reconciliation (v2)

نظام كامل يطابق **دفتر الخزينة** مع **كشف الحساب البنكي** تلقائياً، مع
**مصادقة وصلاحيات، حفظ في قاعدة بيانات، اعتماد المشرف، وسجل تدقيق**.

## التشغيل
```bash
pip install -r requirements.txt
uvicorn main:app --reload
# افتحي http://127.0.0.1:8000
```
تُنشأ قاعدة البيانات `taswiya.db` تلقائياً مع حسابات تجريبية:

| المستخدم | كلمة المرور | الدور | الصلاحيات |
|---|---|---|---|
| admin | admin123 | مدير | كل شيء + سجل التدقيق |
| supervisor | super123 | مشرف | تشغيل + **اعتماد** التسويات |
| entry | entry123 | موظف إدخال | تشغيل التسوية وعرض النتائج |

## المسار في الواجهة
1. تسجيل الدخول.
2. «تسوية جديدة»: رفع دفتر الخزينة + اختيار الشهر + رفع كشف البنك → تشغيل المطابقة (تُحفظ كمسودة).
3. «السجل»: عرض كل التسويات؛ المشرف يفتح تسوية ويضغط **اعتماد**.
4. «سجل التدقيق» (للمدير فقط): كل دخول/تشغيل/اعتماد/تصدير مسجّل بالاسم والوقت.
5. تحميل تقرير Excel ملوّن بضغطة.

## البنية
| الملف | الدور |
|---|---|
| `reconciliation.py` | محرك المطابقة (مستوى الدفعة + كشف التكرار) |
| `database.py` | مخطط SQLite: users, sessions, runs, run_items, audit_log |
| `auth.py` | تجزئة كلمات المرور (PBKDF2)، الجلسات، صلاحيات RBAC |
| `report.py` | تقرير Excel ملوّن |
| `main.py` | خدمة FastAPI (login/sheets/reconcile/runs/approve/audit/report) |
| `static/index.html` | لوحة التحكم العربية (دخول، تسوية، سجل، تدقيق) |

## الأمان (يغطي القسم ٤ من الخطة)
- كلمات المرور مجزّأة بـ PBKDF2-HMAC-SHA256 (٢٠٠ ألف تكرار) ولا تُخزَّن نصاً.
- جلسات برموز عشوائية تنتهي بعد ٨ ساعات.
- صلاحيات هرمية: موظف < مشرف < مدير. الاعتماد للمشرف فأعلى، التدقيق للمدير.
- سجل تدقيق كامل (من فعل ماذا ومتى).

## الانتقال للإنتاج
- **قاعدة البيانات:** نفس المخطط يُهاجَر إلى **SQL Server** (تغيير سلسلة الاتصال والأنواع).
- **الواجهة:** تُعاد كتابة `index.html` كتطبيق **Angular** يستهلك نفس الـAPI.
- **الربط البنكي:** استبدال رفع الملفات بربط مباشر مع API البنك (Open Banking) للتحديث الفوري.
- **التشفير:** نشر خلف HTTPS (TLS) لتشفير النقل.
